\begin{frame}{What is a Programming Language?}
  \begin{columns}
    \column{0.5\textwidth}
    A programming language is a structured way for humans to
    communicate instructions to a machine. Computers fundamentally
    operate on binary logic, which is unintuitive for humans.

    \column{0.5\textwidth}
    \vspace{0.3cm}
    They are essential because they:
    \begin{itemize}
      \item Translate human intent into machine-executable steps
      \item Manage complexity through abstraction
      \item Enable scalable and maintainable systems
    \end{itemize}
  \end{columns}

  \centering
  \includegraphics[width=0.6\textwidth,keepaspectratio]{img/Computer-Program-Compilation-1-1-3821671490.jpg}
\end{frame}

\begin{frame}{Early Concept of an Algorithm}
  \begin{columns}
    \column{0.6\textwidth}
    Long before modern computers, the concept of an algorithm existed
    as a precise sequence of steps to solve a problem.

    \vspace{0.3cm}
    In the 19th century:
    \begin{itemize}
      \item Charles Babbage designed the Analytical Engine
      \item Ada Lovelace described symbolic computation
      \item First algorithm intended for machine execution
    \end{itemize}

    This marked the conceptual birth of programming.

    \column{0.4\textwidth}
    \centering
    \includegraphics[width=0.9\textwidth]{img/ada_babbage.jpg}
  \end{columns}
\end{frame}

\begin{frame}{Assembly Language}
  \begin{columns}
    \column{0.5\textwidth}
    Assembly language was the first step away from raw machine code,
    replacing binary with mnemonic instructions.

    \vspace{0.3cm}

    \column{0.5\textwidth}
    Key characteristics:
    \begin{itemize}
      \item Very close to hardware
      \item Nearly 1:1 with CPU instructions
      \item Efficient but difficult to maintain
      \item Architecture-dependent
    \end{itemize}
  \end{columns}
  \centering
  \includegraphics[width=0.6\textwidth,keepaspectratio]{img/asm.png}
\end{frame}

\begin{frame}{Fortran}
  \begin{columns}
    \column{0.68\textwidth}
    Fortran (FORmula TRANslator) was one of the first high-level
    programming languages, created for scientific computation.
    Developed by John Backus at IBM

    \vspace{0.3cm}
    It introduced:
    \begin{itemize}
      \item Mathematical expressions in code
      \item Compiler abstraction
      \item Efficient numerical processing
    \end{itemize}

    \column{0.28\textwidth}
    \centering
    \includegraphics[width=0.9\textwidth]{img/fortran-for-physics-f90-meme-v0-87ltrusg94cd1.png}
  \end{columns}
\end{frame}

\begin{frame}{COBOL}
  \begin{columns}
    \column{0.68\textwidth}
    COBOL was designed for business and administrative applications.
    It is an imperative, procedural, object-oriented language.
    \vspace{0.3cm}
    Its main ideas:
    \begin{itemize}
      \item English-like syntax
      \item Focus on data processing
      \item Widely used in finance and government
    \end{itemize}

    \column{0.28\textwidth}
    \centering
    \includegraphics[width=0.9\textwidth]{img/one_not_simple_work_with_cobol.png}
  \end{columns}
\end{frame}

\begin{frame}{C Language}
  \begin{columns}
    \column{0.5\textwidth}
    The C language balances low-level control with high-level abstraction.

    \column{0.5\textwidth}
    \vspace{0.3cm}
    It enabled:
    \begin{itemize}
      \item Direct memory manipulation
      \item Portable system programming
      \item Efficient development
    \end{itemize}
  \end{columns}
  \vspace{0.3cm}
  \centering
  \includegraphics[width=0.6\textwidth, keepaspectratio]{img/c.jpg}
\end{frame}

\begin{frame}{Functional programming}
  \begin{columns}
    \column{0.55\textwidth}
    Functional programming is a paradigm where computation is treated as the
    evaluation of mathematical functions. It avoids changing state and mutable
    data, emphasizing immutability. Functions are first-class citizens and can
    be passed, returned, and composed. It encourages pure functions, which
    produce the same output for the same input without side effects.

    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/lambda_hl.png}
    \par\vspace{0.2cm}
    \footnotesize{A Half-Life reference}
  \end{columns}
\end{frame}

\begin{frame}{LISP}
  \begin{columns}
    \column{0.5\textwidth}
    The LISP family of functional programming languages is one of the oldest
    high-level language families, created in the late 1950s.
    Notable dialects include Scheme, Common Lisp, and Clojure.

    \column{0.5\textwidth}
    \vspace{0.3cm}
    Key characteristics:
    \begin{itemize}
      \item Based on symbolic computation and list processing
      \item Uses a simple and uniform syntax (prefix notation)
      \item Code and data share the same structure (homoiconicity)
      \item Widely used in artificial intelligence research
    \end{itemize}
  \end{columns}
  \vspace{0.3cm}
  \centering
  \includegraphics[width=0.5\textwidth,keepaspectratio]{img/lisp.png}
\end{frame}

\begin{frame}{Structure and logic}
  \begin{columns}
    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/structure_and_logic.png}
    \par\vspace{0.2cm}
    \footnotesize{Spaghetti code}

    \column{0.55\textwidth}
    To combat unmanageable "spaghetti code," the industry embraced
    modularity and formal flow control. This era replaced arbitrary
    jumps with nested logic and structured blocks. It proved that
    programming could be a disciplined engineering craft, focused on
    readable, verifiable execution paths.

  \end{columns}
\end{frame}

\begin{frame}{SQL}
  \begin{columns}
    \column{0.5\textwidth}
    Although not strictly a programming language, SQL has been wildy successful
    in the world of relational database administration and querying. It is
    based on the theory of relational calculus and algebra. These foundations
    allow SQL to express complex data operations in a declarative way, focusing
    on what to retrieve rather than how to compute it.

    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/sql.jpg}
    \par\vspace{0.2cm}
    \footnotesize{ALL CAPS}
  \end{columns}
\end{frame}

\begin{frame}{Haskell}
  \begin{columns}
    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/haskell.jpg}
    \par\vspace{0.2cm}
    \footnotesize{The joys of purely functional programming}

    \column{0.55\textwidth}
    It was designed to unify the functional programming community, offering a
    purely functional approach while advancing sophisticated type systems. It
    also introduced lazy evaluation, a feature that was quite unusual at the
    time.
  \end{columns}
\end{frame}

\begin{frame}{Object oriented shift}
  \begin{columns}
    \column{0.55\textwidth}
    Complexity management shifted from procedural steps to
    data-centric encapsulation. By bundling state and behavior into
    objects, developers gained a new way to build and maintain
    massive systems. This paradigm shift redefined how software is
    architected, focusing on modeling real-world entities.

    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/object_oriented_shift.png}
    \par\vspace{0.1cm}
    \footnotesize{Object oriented design patterns}
  \end{columns}
\end{frame}

\begin{frame}{OCaml}
  \begin{columns}
    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/ocaml.jpg}
    \par\vspace{0.2cm}
    \footnotesize{funni comment}

    \column{0.55\textwidth}
    It was designed to extend the ML family by combining functional
    programming with practical features for real-world use. It offers a strong
    static type system with powerful type inference, while also supporting an
    object-oriented style, making it both functional and object-oriented. This
    balance allows it to be expressive and efficient, suitable for both research
    and industrial applications.
  \end{columns}
\end{frame}

\begin{frame}{Scripting and productivity boom}
  \begin{columns}
    \column{0.55\textwidth}
    As the internet scaled, the focus shifted from machine efficiency
    to "developer velocity." High-level scripting and virtual
    machines prioritized human time, allowing for rapid prototyping
    and platform independence. Programming became more about the
    speed of turning ideas into functional software than managing
    hardware constraints.

    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/scripting_and_productivity.png}
    \par\vspace{0.2cm}
    \footnotesize{Node.js programming environment}
  \end{columns}
\end{frame}

\begin{frame}{Contemporary language trends}
  \begin{columns}
    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/rust_pipeline.png}
    \par\vspace{0.2cm}
    \footnotesize{Path of a Rust programmer}

    \column{0.55\textwidth}
    Modern development focuses on "guardrails" eliminating memory
    errors and concurrency bugs at the compiler level. This is also
    an era of refined research; projects like Fram (UWr) are pushing
    the ML tradition forward by using algebraic effects and implicit
    parameters to handle side effects with modular, ergonomic precision.
  \end{columns}
\end{frame}

\begin{frame}{Future of programming}
  \begin{columns}
    \column{0.4\textwidth}
    \centering
    \includegraphics[width=\textwidth]{img/future_of_programming.png}
    \par\vspace{0.2cm}
    \footnotesize{Future of programming with LLMs}

    \column{0.55\textwidth}
    We are moving from "writing" code to "orchestrating" intent. The
    rise of AI agents shifts the programmer’s role toward high-level
    architecture, while the dawn of quantum computing demands an
    entirely new logical framework, shifting our focus from bits and
    states to the complex probabilities of qubits and superposition.
  \end{columns}
\end{frame}
